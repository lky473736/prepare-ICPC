바킹독의 실전 알고리즘 0x03
: 배열
2024. 06. 23.

-----------

1. 배열
- 블록 형태로 기억장소 내에서 연속적으로 저장되는 형태
- index와 값을 1:1 mapping, 같은 데이터형이 저장될 수 있음

2. 배열의 성질
- O(1)로 k번째 원소를 확인하고 변경할 수 있음
    - ex) cout >> arr[1]; 
          arr[1] = 10;
          전부 상수 시간으로 처리 가능
- cache hit ratio가 높음 (CPU가 cache에 있는 데이터를 가져오는 정도)
- overhead가 거의 없음
- 할당에 제약이 걸림

3. 임의의 위치에 원소를 추가하고 삭제
- 끝에 있는 원소를 pop하거나 append한다 -> O(1)
- (임의의) 중간에 있는 위치에 원소를 추가, 삭제 -> O(n)

4. 배열을 초기화하는 방법
- memset : 비추천 
- for문으로 값을 전부 NULL로 변경
- fill

5. vector
- 배열은 크기를 가변할 수 없으나 vector는 가변 가능
- vector는 추후에 해쉬 만들 때 편함
